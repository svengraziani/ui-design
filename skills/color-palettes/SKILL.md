# Color Palettes

## How to Use These Palettes

Each palette provides a cohesive set of **Primary**, **Neutral**, and **Supporting** colors with 10 shades each (dark to light). When building a UI:

1. **Pick one palette** as your foundation
2. **Primary colors** (5-10% of UI): buttons, links, navigation, icons, accent borders, emphasized text
3. **Neutral colors** (80-90% of UI): text, backgrounds, panels, borders, secondary buttons
4. **Supporting colors** (5-10% of UI): semantic states (error, warning, success), accent features, badges

## Color Usage Guidelines

### Primary Colors
- Should appear the most in the UI
- Use for: primary actions, links, navigation items, icons, accent borders, emphasized text
- Pick the 500 shade as the base (good for button backgrounds)
- Use 700-900 for text on colored backgrounds
- Use 50-200 for tinted backgrounds (alerts, badges, selected states)

### Neutral Colors
- Make up the majority of the UI (80-90%)
- Use for: text, backgrounds, panels, borders, secondary buttons, form controls
- 900: darkest text (headings)
- 700: body text
- 500: secondary text, icons
- 300: borders, dividers
- 100: subtle backgrounds
- 50: page backgrounds

### Supporting Colors
- Use sparingly to avoid overpowering primary colors
- Red: errors, destructive actions, negative trends
- Yellow/Orange: warnings, pending states
- Green/Teal: success, positive trends, completed states
- Blue: informational, links (when primary isn't blue)
